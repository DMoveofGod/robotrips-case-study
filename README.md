Robotrips — Global Travel Planner & Robo-Taxi Comparison Platform

Role: Freelance Full-Stack Developer
Client: Simon Whittle, Founder
Stack: Base44 · DNS Configuration
site: https://robotrips.com/


The Problem

Robotrips needed to go from a working build to a fully live, production-accessible platform at robotrips.com — but a DNS configuration conflict on GoDaddy was blocking the domain from resolving correctly, keeping the platform effectively invisible to users.

What I Did


Diagnosed the DNS configuration conflict blocking the custom domain from pointing to the live Base44-hosted platform
Resolved the GoDaddy DNS setup to correctly route robotrips.com to production
Verified the platform was fully live and accessible post-fix, rather than relying on the platform's own "deployed" status


Outcome

robotrips.com went from inaccessible to fully live in production, unblocking the founder's ability to share and promote the platform publicly.

Why This Matters

DNS and domain routing issues are a common but under-discussed blocker for no-code platforms like Base44 — a build can be "finished" on the platform side and still be completely unreachable to real users. Diagnosing and fixing infrastructure issues like this, not just building features, is part of getting a client's product genuinely live.
